# Export Learning - 导出并保存学习记录

## 描述
将当前学习内容导出到 learning-records 目录，并提交到 Git 仓库。

## 使用场景
- 每次学习结束后
- 需要保存学习笔记时
- 完成一个学习模块后

## 执行步骤

### 第一步：询问用户学习主题
询问用户：今天学习的主题是什么？

### 第二步：检查/创建记录文件
1. 检查 `learning-records` 目录是否存在
2. 如果不存在，创建该目录
3. 生成文件名格式：`learning-records/YYYY-MM-DD-主题.md`

### 第三步：询问导出内容
询问用户想要导出什么：
- A) 完整学习笔记（包含所有讨论内容）
- B) 今日总结（核心知识点+一句话总结）
- C) 自定义（用户指定）

### 第四步：写入文件
将内容写入对应文件

### 第五步：Git 操作
检查是否为 Git 仓库：
- 如果不是：询问是否初始化 Git 仓库
  - 如果用户同意：执行 `git init`
  - 如果用户不同意：仅保存文件，不提交
- 如果是 Git 仓库：
  1. `git add .`
  2. `git commit -m "feat: 添加 [主题] 学习记录"`
  3. 检查是否有远程仓库
  4. 如果有远程仓库：执行 `git push`
  5. 如果没有远程仓库：告知用户"已保存到本地，需要手动添加远程仓库"

### 第六步：确认完成
告知用户：
- 文件保存位置
- Git 提交状态
- 远程推送状态（如有）

---

## 文件格式模板

```markdown
# [学习主题]

> 学习日期：[YYYY-MM-DD]
> 学习目的：[用户说明的目的]

---

## 学习内容

[根据对话内容整理]

---

## 核心知识点

| 概念 | 理解 |
|------|------|
| ... | ... |

---

## 一句话总结

> [用一句话概括今天学到的最重要的东西]

---

## 下次学习预告

- [ ] 待学内容1
- [ ] 待学内容2

---

## 相关文件

- 上次学习：[文件名]
- 下次学习：[文件名]

---

*由 Claude Code 学习助手生成*
```

  C) 自定义
