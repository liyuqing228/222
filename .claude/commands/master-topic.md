# Master Any Topic - 快速掌握任意主题

## 描述
一个基于学习理论（建构主义、检索练习、间隔重复）的结构化学习框架。通过苏格拉底式提问和迭代式对话，帮助你在最短时间内建立对任意主题的深度理解。

## 输入
- 主题名称：用户想要学习的主题
- 当前理解程度：（可选）初学者/有一些了解/基本掌握

---

## 学习框架

### 第一阶段：建立框架（10分钟）

**目标**：快速摸清主题的边界和结构

用这些问题快速探索：

1. "关于【主题】，普通人最常问的3个问题是什么？"
2. "这个领域最核心的5个概念是什么？用一句话解释每个。"
3. "如果要向一个10岁小孩解释这个主题，你会怎么讲？"
4. "这个主题可以拆分成哪几个子主题/模块？"

**输出**：主题概览图 + 核心概念清单

---

### 第二阶段：深度建构（15分钟）

**目标**：针对每个核心概念深入理解

对每个核心概念，依次回答：

1. **定义**："这个概念的核心定义是什么？"
2. **为什么**："为什么这个概念重要？解决什么问题？"
3. **类比**："能否用一个生活中的类比来解释？"
4. **例子**："最典型的应用场景/例子是什么？"
5. **边界**："这个概念的局限性或常见误解是什么？"

**技巧**：每学完一个概念，立即：
- "用自己的话复述一遍"
- "这个概念和【已知的XX】有什么联系？"

---

### 第三阶段：强制检索（10分钟）

**目标**：通过输出强化记忆和理解

1. **思维导图**：用ASCII或结构化文本画出这个主题的知识图谱
2. **电梯演讲**：如果只有60秒，你会如何向一个完全不懂的人介绍这个主题？
3. **提出问题**：列出你仍然存在的3个疑惑
4. **实践想法**：你能马上做一件什么小事来应用这个知识？

---

### 第四阶段：复盘与间隔（5分钟）

**目标**：巩固学习成果，设计复习计划

1. **今天学到的最重要的一件事是什么？**
2. **如果要用一句话总结整个主题，你会怎么说？**
3. **设计你的复习计划**：
   - 明天：重读核心概念清单
   - 3天后：用自己的话写一段总结
   - 一周后：教给别人（或向AI复述）

---

## 执行流程

1. **开始时**：询问用户要学习的主题和当前程度
2. **第一阶段**：并行执行4个探索问题（除非用户时间有限，只问最重要的1-2个）
3. **第二阶段**：选择2-3个最核心的概念进行深度挖掘
4. **第三阶段**：要求用户完成检索练习
5. **结束时**：输出学习总结和复习提醒

## 关键原则

- **先框架，后细节** - 不要一上来就陷入细节
- **强制输出** - 每学完一个部分必须有输出，不能只是被动听
- **联系旧知** - 不断关联用户已知的知识
- **保持好奇** - 承认未知，列出后续探索路径

## 时间提醒

- 如果时间有限：跳过第一阶段的探索，直接进入第二阶段深挖最重要的2个概念
- 如果时间充裕：可以深入第三阶段的每个练习
- 标准流程约30-40分钟，但可以压缩到10分钟

## 示例对话

```
用户：我想了解区块链
助手：太好了！让我们用20分钟快速掌握区块链的核心。

首先，告诉我：你对区块链现在的理解程度是？
A) 完全小白
B) 听过一些名词但不理解
C) 了解基本原理但想深入

（用户选择B）

好，让我们开始：

【第一阶段 - 探索】
1. 关于区块链，人们最常问的3个问题是...
2. 区块链最核心的5个概念是...
3. 用生活类比的话...

（继续...）
```
